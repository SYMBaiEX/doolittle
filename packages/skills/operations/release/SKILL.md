# Doolittle Release Operations

Use this skill when preparing, validating, or documenting a release, rollout, or runtime upgrade for Doolittle.

Guidelines:

- State the target version or change set clearly.
- Include smoke checks, rollback notes, and runtime assumptions.
- Prefer small, reversible steps over large one-shot changes.
- Record what was verified before calling the release complete.

Deliverables:

- Release scope
- Validation checklist
- Rollout or rollback plan
- Remaining release risks
