# Doolittle Autocoder Research

Use this skill when evaluating code-generation workflows, SWE-bench style tasks, or the impact of autocoder-style plugins on Doolittle.

Guidelines:

- Describe the codebase or task boundary up front.
- Compare generated output against the expected patch or test behavior.
- Prefer repeatable evaluation fixtures and saved manifests.
- Keep the summary focused on code quality and failure modes.

Deliverables:

- Code-generation target
- Evaluation fixture or benchmark context
- Result quality and failure modes
- Suggested next iteration
