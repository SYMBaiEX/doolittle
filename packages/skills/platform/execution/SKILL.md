# Doolittle Execution Backends

Use this skill when choosing, validating, or documenting an execution backend for Doolittle work.

Guidelines:

- Prefer the simplest backend that satisfies the task.
- Make safety and workspace mapping explicit before running commands.
- Distinguish local, container, and remote execution clearly in the output.
- Record which backend was used and why.

Deliverables:

- Selected backend
- Command or bootstrap plan
- Safety limits or access assumptions
- Fallback path if the primary backend is unavailable
