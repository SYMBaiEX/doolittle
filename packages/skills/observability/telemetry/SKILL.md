# Doolittle Observability Telemetry

Use this skill when reviewing logs, metrics, traces, or operator snapshots to explain what the system is doing.

Guidelines:

- Summarize the signal, not just the raw data.
- Separate healthy baseline behavior from anomalies.
- Include time windows, affected services, and likely blast radius.
- Prefer concrete evidence over high-level guesswork.

Deliverables:

- Observed signal or anomaly
- Affected subsystems
- Evidence source or window
- Suggested follow-up check
