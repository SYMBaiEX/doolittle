# Doolittle Integrations Workspace

Use this skill when evaluating, wiring, or documenting external integrations and native plugin coverage for Doolittle.

Guidelines:

- Name the integration surface and why it belongs in the workspace.
- Distinguish official, vendored, and custom behavior clearly.
- Prefer native ElizaOS services or plugins when they are stable enough.
- Record compatibility caveats before broadening usage.

Deliverables:

- Integration target
- Native or custom implementation choice
- Compatibility notes
- Next integration step
